Скрипт не запускается, а в консоли — красное `Uncaught SyntaxError`. Или того
хуже: страница молча не реагирует, и непонятно, где сломалось. Чаще всего причина
банальна — пропущенная скобка или лишняя запятая. Разберём, как быстро найти
такую ошибку.

## Как проверить JavaScript на ошибки

Самый быстрый способ — [проверить JavaScript онлайн](/tools/javascript-syntax-checker):
вставьте код, и парсер укажет строку и колонку, где споткнулся разбор. Это
удобно, когда нужно проверить чужой сниппет, кусок кода с телефона или просто
получить второе мнение, не открывая IDE.

Локально то же самое умеет **консоль браузера** (F12 → Console): вставьте код,
нажмите Enter — движок сразу сообщит о синтаксической ошибке. А в редакторе за
этим следит линтер (ESLint) — подчёркивает проблему прямо по мере набора.

Важно понимать, что все эти инструменты ловят **синтаксис** — грамматику языка.
Логические ошибки (код валиден, но делает не то) они не видят.

## Частые синтаксические ошибки

Больше девяти из десяти «поломок» — это одно из следующего.

**Незакрытая скобка или кавычка.** Пропустили `}`, `)` или `"` — и парсер доходит
до конца файла, не поняв структуру:

```js
function greet(name {
  return "Привет, " + name
}
```

Здесь нет `)` после `name` — `Unexpected token '{'`.

**Лишняя или недостающая запятая** в объекте и массиве:

```js
const user = {
  name: "Дмитрий",
  age: 30,
  city: "Москва",   // висячая запятая в объекте допустима,
}                     // а вот пропуск запятой между полями — ошибка
```

**Опечатка в ключевом слове:** `fucntion`, `retrun`, `cosnt` — движок не узнаёт
слово и падает.

**Спутанные `=` и `==`:** `if (x = 5)` вместо `if (x === 5)` — это не ошибка
синтаксиса, код запустится, но сделает присваивание вместо сравнения. Такое
ловит уже линтер, а не парсер.

**Забытые кавычки у строки:** `const city = Москва` — движок примет `Москва` за
имя переменной, которой нет.

## Синтаксис или выполнение

Ошибки в JavaScript делятся на два вида, и это важно различать:

- **Синтаксические** (`SyntaxError`) — код не соответствует грамматике языка.
  Скрипт **не запускается вообще**. Их и ловит проверка синтаксиса — до запуска,
  просто разбирая текст.
- **Ошибки выполнения** (`ReferenceError`, `TypeError` и другие) — код валиден и
  запустился, но упал уже в работе: обратились к несуществующей переменной,
  вызвали метод у `undefined`. Их видно только в рантайме, в консоли браузера.

Проверка синтаксиса закрывает первый случай — тот, где IDE ещё не открыта, а
понять «почему вообще не грузится» нужно быстро.

## А что с TypeScript

TypeScript — это тот же JavaScript плюс **типы**: аннотации вроде `name: string`,
интерфейсы, дженерики. Обычная проверка JS на таком коде споткнётся уже на
двоеточии с типом — для JavaScript это чужой синтаксис. TypeScript проверяет свой
компилятор `tsc` (или онлайн-песочница TypeScript Playground): он смотрит не
только грамматику, но и сходятся ли типы. Наш инструмент рассчитан на JavaScript и
JSX — то, что выполняется в браузере напрямую; для проверки типов TS используйте
`tsc`.

## Коротко

- Синтаксическую ошибку в JS ищите по строке и колонке из сообщения парсера —
  проверьте код онлайн, в консоли браузера (F12) или через ESLint.
- Топ причин: незакрытая скобка/кавычка, пропущенная запятая, опечатка в ключевом
  слове.
- `SyntaxError` не даёт коду запуститься; `TypeError`/`ReferenceError` возникают
  уже в рантайме — это разные вещи.
- Типы TypeScript обычная JS-проверка не видит — для них нужен `tsc`.
